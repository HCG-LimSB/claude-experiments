# 배포 규칙

## 커밋 요청 시

커밋 관련 명령(예: "커밋해줘", "commit")을 받으면:

1. 현재 브랜치가 `main`이면:
   - 작업 내용을 바탕으로 적절한 이름의 새 브랜치를 생성한다.
   - 변경 사항을 새 브랜치로 옮겨 커밋한다.
2. 현재 브랜치가 `main`이 아니면: 해당 브랜치에 바로 커밋한다.
3. 커밋 메시지는 변경 내용을 기반으로 자동 작성한다.

`main`은 항상 스쿼시 머지만 허용한다 — `main`에 직접 커밋하지 않는다.

## 배포 준비 시

"배포 준비"라는 말을 들으면:

0. **브랜치 상태 먼저 확인한다:**
   - 현재 브랜치가 이미 머지된 브랜치인지 확인 (`gh pr list --head <branch> --state merged`)
   - 머지된 브랜치라면:
     1. `main`을 최신화한다 (`git checkout main && git pull`)
     2. 작업 내용을 바탕으로 새 브랜치를 생성한다
     3. 변경 사항을 새 브랜치에 커밋한다
     4. 새 브랜치로 PR을 생성한다
1. 현재 브랜치에 열린 PR이 없으면 → 새 PR을 생성한다.
2. 이미 PR이 존재하면 → 현재 작업 내용을 반영해 PR 제목과 본문을 최신화한다.
3. PR 본문에는 변경 사항 요약과 테스트 계획을 포함한다.
